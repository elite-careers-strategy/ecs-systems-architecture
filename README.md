# ECS Systems Architecture

Public-safe architecture and operating principles for Elite Careers Strategy.

Elite Careers Strategy is a founder-led advisory firm for candidates targeting selective outcomes in corporate law, investment banking, private equity, venture capital and consulting.

This repository documents the public version of the ECS operating model: how demand is captured, how candidate gaps are diagnosed, how support is structured, and how proof and privacy are governed.

## What this repo is

A public-safe overview of the ECS system:

- Architecture principles
- Intake-to-engagement flow
- Operating-system design
- Proof governance
- Publication boundaries

## What this repo is not

This repo does not contain:

- Client identities
- Private proof records
- Internal pricing
- Internal prompts
- Database IDs
- Credentials or configuration values
- Private operational runbooks

## Architecture summary

The ECS system has five public-safe layers:

1. Public surface - website, articles and public explainers.
2. Intake and qualification - structured capture of stage, target, timeline and decision-maker context.
3. Diagnostic and engagement design - diagnosis before prescription.
4. Operating system - tasks, readiness, application pipeline and feedback loops.
5. Proof and privacy governance - consent-safe proof deployment and claim control.

## Operating principle

The system is designed around one constraint: founder judgement should be reserved for the work where founder judgement is decisive.

Everything else should become a repeatable process, a template, a checklist, a workflow or an agent-supported task.
