# Proof Governance

Proof is one of the most important assets in the ECS system.

It is also one of the highest-risk areas if handled casually. This document sets the public-safe principles for how proof should be governed.

## Core rule

A proof item should only be used publicly when the identity, claim and context are all safe.

That means:

- The claim is accurate.
- The evidence exists.
- The deployment context is appropriate.
- Consent has been confirmed where naming is involved.
- The proof does not reveal private family, client or candidate information.

## Proof categories

### Public-safe proof

Can be used in public materials after review.

Examples:

- Aggregate outcomes
- Public testimonials with permission
- Public speaking or press references
- Sanitised process descriptions

### Private proof

Can be used only in controlled private contexts.

Examples:

- Anonymised family evidence
- Private client outcomes
- Documentary proof kept on file
- Sensitive screenshots or correspondence

### Internal-only proof

Should not be used outside ECS operations.

Examples:

- Raw evidence files
- Internal notes
- Consent status records
- Unreviewed proof items
- Private relationship context

## Claim control

Every public claim should pass four questions:

1. Is it true?
2. Is it evidenced?
3. Is it consent-safe?
4. Is it necessary for the audience?

If the answer to any question is no, the claim should not be published.

## Naming rule

Do not name a client, family, student, candidate or outcome unless naming is explicitly approved.

Default position: anonymise.

## Public architecture rule

A public systems repo can explain how proof governance works. It should not contain the proof itself.

This repo should never contain:

- Raw screenshots of client evidence
- Candidate names
- Family names
- Private emails or messages
- Outcome evidence files
- Consent records

## Why this matters

Trust compounds slowly and can be damaged quickly.

The purpose of proof governance is to let ECS demonstrate credibility without creating privacy risk.
